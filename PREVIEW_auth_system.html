Como o sistema funcionará: Ao abrir qualquer página do sistema GR7, o colaborador verá uma tela de login. Ele seleciona seu nome na lista e digita seu PIN de 4 dígitos. O sistema valida e libera o acesso conforme o perfil: Administrador tem acesso total, Colaborador tem acesso restrito às suas funções.

Segurança da sessão

Como a segurança funciona:
  • PIN de 4 dígitos armazenado como hash SHA-256 — ninguém vê o PIN real
  • Sessão salva em sessionStorage — expira ao fechar o navegador
  • Sessão expira automaticamente após 8 horas de inatividade
  • 3 tentativas erradas de PIN bloqueiam o acesso por 30 segundos
  • Ao clicar "Sair" a sessão é destruída imediatamente

Como as restrições aparecem na tela

Botões restritos ficam ocultos ou desabilitados para colaboradores. Se tentar acessar uma URL restrita diretamente, é redirecionado para o login com uma mensagem de acesso negado. Não há mensagem de erro confusa — apenas o fluxo correto.
Mudança no cadastro de colaboradores: O modal de cadastro ganhará dois campos novos: Perfil (Admin ou Colaborador) e PIN de acesso (4 dígitos). O administrador define o PIN ao criar o usuário; o colaborador usa esse PIN para entrar.

👥 Equipe · 3 membros
Alterar PIN: O administrador pode editar qualquer colaborador e definir um novo PIN a qualquer momento. O colaborador não consegue alterar o próprio PIN.
Alterações no banco de dados: São necessárias apenas 2 colunas novas na tabela colaboradores no Supabase. Execute o SQL abaixo no SQL Editor do seu projeto Supabase. Nenhuma outra tabela é alterada.

SQL para executar no Supabase (SQL Editor)

-- ═══════════════════════════════════════════════════
-- GR7 — Adicionar autenticação por PIN à tabela colaboradores
-- Execute este bloco no Supabase SQL Editor
-- ═══════════════════════════════════════════════════

-- 1. Adicionar coluna de perfil (admin ou colaborador)
ALTER TABLE colaboradores
  ADD COLUMN IF NOT EXISTS perfil TEXT
  NOT NULL DEFAULT 'colaborador'
  CHECK (perfil IN ('admin', 'colaborador'));

-- 2. Adicionar coluna de PIN (hash SHA-256, 64 caracteres)
ALTER TABLE colaboradores
  ADD COLUMN IF NOT EXISTS pin_hash TEXT;

-- 3. (Opcional) Definir PIN padrão para colaboradores existentes
--    PIN padrão: 1234  →  hash SHA-256 abaixo
UPDATE colaboradores
  SET pin_hash = '03ac674216f3e15c761ee1a5e255f067953623c8b388b4459e13f978d7c846f4'
  WHERE pin_hash IS NULL;

-- 4. Verificar resultado
SELECT id, nome, cargo, perfil,
       CASE WHEN pin_hash IS NOT NULL THEN '✅ PIN definido'
            ELSE '⚠️ Sem PIN' END AS status_pin
FROM colaboradores
ORDER BY id;
